Roadmap IT Network

Peta jalan belajar networking dari pemula hingga profesional.

‚ÑπÔ∏è
Estimasi Waktu

Waktu belajar tiap level tergantung konsistensi. Rata-rata 1-3 bulan per level dengan belajar 1-2 jam/hari.

Level 1 ‚Äî Fondasi (1-3 Bulan)

  • Dasar Jaringan Komputer ‚Äì OSI Model, TCP/IP, IP Address
  • Subnetting ‚Äì Perhitungan subnet, CIDR, VLSM
  • Perangkat Jaringan ‚Äì Switch, Router, Hub, Access Point
  • Cisco Packet Tracer ‚Äì Instalasi dan penggunaan dasar

Target Sertifikasi: Cisco CCNA (200-301)

Level 2 ‚Äî Intermediate (3-6 Bulan)

  • VLAN & Trunking ‚Äì Segmentasi jaringan
  • Spanning Tree Protocol (STP)
  • Routing Protokol ‚Äì OSPF, EIGRP, BGP dasar
  • NAT & PAT, ACL
  • PNETLab ‚Äì Lab dengan IOS virtual

Target Sertifikasi: Cisco CCNP Enterprise

Level 3 ‚Äî Advanced (6-12 Bulan)

  • BGP Advanced ‚Äì iBGP, eBGP, Policy Routing
  • MPLS ‚Äì Multi-Protocol Label Switching
  • Network Automation ‚Äì Python, Ansible
  • Network Security ‚Äì Firewall, IPS/IDS, VPN

Target Sertifikasi: Cisco CCNP SP / CCIE

Instalasi Cisco Packet Tracer

Cara download dan install Cisco Packet Tracer di Windows, macOS, dan Linux.

packet-tracerinstalasipemula

Langkah-langkah Instalasi

1

Buat Akun Cisco NetAcad

Buka netacad.com ‚Üí klik Sign Up ‚Üí isi form dengan email aktif ‚Üí verifikasi email.

2

Download Packet Tracer

Login ke NetAcad ‚Üí klik Resources ‚Üí Download Packet Tracer ‚Üí pilih versi sesuai OS.

3

Install Aplikasi

Windows: jalankan .exe sebagai Administrator. Ubuntu: sudo dpkg -i CiscoPacketTracer_*.deb

4

Login di Aplikasi

Buka Packet Tracer ‚Üí pilih Login with Cisco NetAcad ‚Üí masukkan email dan password.

Instalasi Per Platform

  1. Jalankan file .exe yang sudah didownload
  2. Klik Next ‚Üí I Agree ‚Üí Next
  3. Pilih folder instalasi (biarkan default)
  4. Klik Install ‚Üí tunggu selesai ‚Üí Finish

# Install package

sudo apt-get install -f

Konfigurasi VLAN

Implementasi VLAN dan Trunking di switch Cisco.

vlanswitching

Apa itu VLAN?

VLAN (Virtual Local Area Network) adalah teknik membagi satu physical switch menjadi beberapa logical network yang terisolasi satu sama lain.

üí°
Manfaat VLAN

Keamanan jaringan lebih baik, broadcast domain lebih kecil, manajemen lebih fleksibel.

Konfigurasi VLAN di Switch

! Masuk ke mode konfigurasi
enable
configure terminal

! Buat VLAN
vlan 10
 name DATA
vlan 20
 name VOICE
vlan 99
 name MANAGEMENT

! Verifikasi VLAN
do show vlan brief

Assign Port ke VLAN (Access Port)

! PC ke VLAN 10
interface FastEthernet0/1
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast

! Multiple port sekaligus
interface range FastEthernet0/1-10
 switchport mode access
 switchport access vlan 10

Konfigurasi Trunk Port

interface GigabitEthernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,99

! Verifikasi trunk
show interfaces trunk

Routing Statis

Konfigurasi routing manual antar jaringan menggunakan static route.

Konsep Static Route

Routing statis adalah konfigurasi route secara manual oleh administrator. Cocok untuk jaringan kecil dengan topologi yang jarang berubah.

! Static route ke jaringan 192.168.2.0/24 via next-hop
ip route 192.168.2.0 255.255.255.0 192.168.1.2

! Default route (semua traffic ke internet)
ip route 0.0.0.0 0.0.0.0 203.0.113.1

! Verifikasi routing table
show ip route
show ip route 192.168.2.0
‚ö†Ô∏è
Return Route

Jangan lupa konfigurasi route kembali (return route) di router tujuan, agar komunikasi dua arah bisa berjalan.

Tidak Bisa Ping

Diagnosa dan solusi ketika perangkat tidak bisa saling berkomunikasi.

troubleshootingping

Checklist Troubleshooting

1

Cek Fisik (Layer 1)

Pastikan kabel terhubung dan LED port menyala. Jalankan show interfaces Gi0/0 ‚Äî pastikan status up/up.

2

Cek IP Address (Layer 3)

Jalankan show ip interface brief. Pastikan IP address, subnet mask, dan status interface sudah benar.

3

Cek Routing Table

Jalankan show ip route. Pastikan ada route ke jaringan tujuan. Jika tidak ada, tambahkan static route atau cek routing protokol.

4

Cek ACL

Jalankan show ip access-lists. Periksa apakah ada ACL yang memblokir traffic.

Penyebab Umum & Solusi

GejalaKemungkinan PenyebabSolusi
Interface down/downMasalah fisik, kabel rusakGanti kabel, cek port
Interface admin downInterface di-shutdownno shutdown
Ping ke gateway gagalIP tidak se-subnetKoreksi IP address
Ping gateway OK, tapi ke remote gagalTidak ada routeTambah static/dynamic route
Ping intermittentDuplex mismatch, kabel burukSet duplex manual, ganti kabel

Return Route

Sering terlupakan: router di ujung juga harus punya route kembali ke sumber. Cek routing table di kedua sisi!
